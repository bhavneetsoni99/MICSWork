                                    Daniel Mannino • 2 years ago

                                    good article but I have some comments:

                                    - sharding automatically it is not a good idea; rows/columns from tables that are accessed together must be stored in the same shard otherwise you risk huge network traffic problems;

                                    - database normalization makes the data fit for any use; put the data together according to an access pattern makes the data fit only for that particular pattern; in addition, having data in different table does not automatically mean storing data in different files; in Oracle, for example, you can store data from different tables in a cluster ( from the Oracle doc "cluster is a schema object that contains data from one or more tables, all of which have one or more columns in common. Oracle Database stores together all the rows from all the tables that share the same cluster key")

                                    - Teradata is an MPP database, it provides sharding without loosing querying, transactions, and consistency; the same is true for Oracle Exadata;
